# Database Family Playbooks

These playbooks sit one level above the per-database onboarding docs.

Use them when the main question is not "how do I stage this one database?" but instead:

- how should this whole database family be governed
- which modules are siblings versus core versions
- how should future versions be added without muddying semantics
